# RecallMap — Gemma 4 Good Hackathon

## 專案定位

**Tagline**：*Find what you don't know — no AI experience needed.*

**核心使用者**：大學生（自主學習時間最多、第一次需要完全自己管理學習節奏）

**解決的問題**

- 讀完就忘、不知道自己哪裡沒懂
- 不知道怎麼規劃複習時間
- 對 AI 陌生、不知道從哪裡開始

**比賽資訊**：Gemma 4 Good Hackathon（Kaggle × Google DeepMind）｜獎金池 $200,000 美元｜截止日 2026/05/18

---

## 差異化四層

1. 從「複習」升級到「找出你不知道自己不懂的地方」
2. ChatGPT／Gemini 對話紀錄 → 結構化學習資產（唯一在做這件事的工具）
3. 零門檻：不需要懂 AI，拖曳丟進來就好
4. **理解 ≠ 記憶，分開追蹤**：市面上的間隔複習工具（Anki 等）只追蹤「記不記得」。RecallMap 同時追蹤「懂不懂」與「記不記得」兩條軸，懂了才值得記，真正掌握才標綠

---

## 為什麼一定要 Gemma 4

- **多模態**：直接看懂手寫圖片、iPad 匯出筆記
- **Function calling**：串接 Google Calendar、Notion API
- **本地部署（隱私優先）**：筆記與對話紀錄不需上傳給第三方
- **混合架構**：邊緣裝置（E4B）處理敏感資料，雲端（26B/31B）處理複雜推理
- **Agentic workflow**：主動分析、排程、追蹤，不只是出題

## 輸入層

> **⚠️ 2026-04-09 更新**：Gemini 一般帳號有分享連結功能（`gemini.google.com/share/...`），**限制的是學校或企業 Workspace 管理員帳號**。因此 Gemini 分享連結與 ChatGPT 相同，同樣支援兩種輸入方式：官方 Takeout 匯出（歷史舊對話）+ 分享連結即時爬取（單篇新對話）。前端 UI 需在輸入框下方加註：「如果你是用學校帳號登入 Gemini，可能看不到分享按鈕，請改用 Google Takeout 匯出。」
> 

| 來源 | 格式 | 處理方式 |
| --- | --- | --- |
| 教授講義 | PDF、PPT、Word | 文字萃取（**markitdown** → 直接轉為 .md，不需自寫 parser） |
| Notion 筆記 | API 讀取 | 直接串接 |
| iPad 手寫筆記 | 匯出圖片／PDF | Gemma 4 多模態辨識 |
| 紙本手寫筆記 | 手機拍照 | Gemma 4 多模態辨識 |
| ChatGPT 對話紀錄 | 匯出 JSON | 自動解析學習模式 |
| Gemini 對話紀錄 | Google Takeout | 自動解析學習模式 |

## 核心流程

```jsx
丟入學習材料（任何格式）
或連動 ChatGPT / Gemini 對話歷史
        ↓
Gemma 4 理解內容 + 分析對話紀錄
        ↓
自動歸類到知識主題（Python / 線代 / 英文 / 論文...）
先查知識庫有無相似主題，有則合併，無則新建
        ↓
找出知識盲點（你反覆問、你沒有追問、你寫了但沒真正懂）
        ↓
生成「學習地圖」
  🔴 盲點區（理解 < 0.4）— 需要重新學
  🟡 模糊區（理解 0.4–0.7）— 理解中
  🟠 需要複習（理解 > 0.7，記憶衰減中）— 懂但快忘
  🟢 已知區（理解 > 0.7，記憶穩固）— 真正掌握
        ↓
【理解軸】針對 🔴🟡 出理解型題目
  完全不懂 → 先填空/選擇確認知識點，再出簡答
  部分理解 → 簡答為主，要求舉例或類比
  接近掌握 → 換情境應用題，驗證能否 transfer
  ▸ 題目設計確保無法靠剛讀完的短期記憶作答（流暢幻覺防護）
  ▸ 當天 deep 回答只升分，隔天延遲確認才真正升級
        ↓
【記憶軸】針對 🟠 出記憶型題目
  SM-2 間隔複習演算法決定出題時機
  同主題相關概念交錯出題（Interleaving）
  記憶測驗未通過 → 理解分數輕微連動衰減
        ↓
更新學習地圖 → 結合行程空檔 → 自動排入 Google Calendar
        ↓
時間到 → 針對弱點主題複習 → 再次確認
```

## Demo 展示情境

1. **程式設計**（Notion 筆記）→ 找出遞迴盲點 → 學習地圖顯示 🔴 盲點區 → 先出填空確認知識點、再出「舉生活例子說明遞迴」的簡答 → 隔天延遲確認後升級為 🟡
2. **英文語言學習**（手寫筆記拍照）→ Gemma 4 圖片辨識 → 學習地圖顯示 🟠 需要複習 → 交錯出單字填空（Interleaving，5 個單字混合出題）
3. **ChatGPT 對話紀錄**（JSON 匯入）→ 生成學習地圖 → 發現「問過 4 次還沒懂」的概念標為 🔴 → 出「換情境」應用題 → 答對後設「✳️ 初步確認，明天再來驗證」

**魔法時刻 1**：上傳 ChatGPT 對話紀錄 + 說一句「我下週五要考試」→ 30 秒後 Google Calendar 自動出現複習事件

**魔法時刻 2**：上傳一學期的 Gemini 對話記錄 → AI 自動整理出「你問過 12 次遞迴、8 次線代、5 次機率」→ 按熟悉度排序出複習清單，馬上出第一道題

**魔法時刻 3**：貼上今天和 ChatGPT 的對話分享連結 → 10 秒後自動補充進知識庫，發現你在遞迴又問了一個新的盲點

## 技術堆疊

| 層次 | 技術 |
| --- | --- |
| 邊緣模型 | Gemma 4 E4B（Ollama） |
| 雲端模型 | Gemma 4 26B / 31B（Vertex AI） |
| 後端 | Python + FastAPI |
| 前端 | Expo（React Native）→ iOS / Android / Web；Tauri → 桌面 |
| 行程串接 | Google Calendar API |
| 筆記串接 | Notion API |
| 複習演算法 | SM-2（Spaced Repetition）＋ Delayed Confirmation Gate ＋ Interleaving |
| 文件轉換 | **markitdown**（Microsoft 開源）— PDF / PPT / Word → Markdown，統一餵入 LLM |
| 訓練平台 | Google Colab / Vertex AI |

## 已知缺陷與對策

| 缺陷 | 對策 |
| --- | --- |
| 蘇格拉底追問難度過高 | 改以自適應測驗（簡答 → 選擇 → 是非 → 填空）取代，四種題型梯度已能覆蓋不同理解深度，且實作複雜度低、評分邏輯明確 |
| 筆記品質決定出題品質 | AI 先評估筆記完整度，主動補充缺漏再出題 |
| 行程整合太理想化 | 降級為「建議」，讓使用者一鍵確認後再寫入 Calendar |
| ChatGPT 匯出有摩擦 | Onboarding 有逐步引導，說明怎麼匯出 |
| 功能太多做不完 | Demo 聚焦：深做「輸入 → 盲點偵測 → 學習地圖」，行程整合列為加分 |
