# P0-2｜學習地圖 Force Refresh

# 功能描述

目前學習地圖第一次分析後會永久快取，上傳新材料也無法重新分析。加入 `force` 參數讓使用者可以主動觸發重新分析。

---

# 輸入 / 輸出

|  | 說明 |
| --- | --- |
| **輸入** | `GET /v1/sessions/{id}/map?force=true` |
| **輸出** | 重新跑盲點偵測，回傳更新後的學習地圖 |
| **輸入（原有）** | `GET /v1/sessions/{id}/map`（不帶 force） |
| **輸出（原有）** | 行為不變，有快取直接回傳 |

---

# 實作範圍

## ✅ 包含

- `GET /v1/sessions/{id}/map` 加入 `force: bool = False` query 參數
- `force=true` 時清掉 `sess["learning_map"]` 和 `sess["blind_spots"]`，重新跑偵測
- `force=true` 同時清掉所有 `dialogue_sessions`（舊的盲點 ID 失效）
- 回傳格式與現有相同，不增加新欄位

## ❌ 不包含

- 增量更新（只分析新加的 chunks）
- 比較新舊地圖的差異
- 前端的觸發按鈕（前端另外實作）

---

# 驗收標準

- [ ]  不帶 `force` 參數：有快取時直接回傳，不重新呼叫 AI
- [ ]  `?force=true`：重新跑盲點偵測，回傳新結果
- [ ]  `?force=true` 後，舊的 `dialogue_sessions` 被清空
- [ ]  空 chunks 時加 `force=true` 仍回傳 422
