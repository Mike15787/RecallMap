# 專案目標

**一句話**：讓大學生把任何學習材料丟進來，30 秒內找出知識盲點並自動排入複習行程。

**截止日**：2026/05/18（Gemma 4 Good Hackathon）

**MVP 核心流程**：上傳材料 → 盲點偵測 → 學習地圖 → 自適應測驗（簡答 → 選擇 → 是非 → 填空）→ Google Calendar 排程

# 已知架構問題（需修復）

1. **Session 記憶體儲存** — 伺服器重啟所有資料消失，Demo 前必須解決（SQLite 最快）
2. **學習地圖無法重新分析** — 第一次分析後永遠快取，加更多材料無效，需要 `force_refresh` 參數
3. **OAuth state 無持久化** — 多 worker 環境下 state 會跨 worker 找不到，pending state 也沒有過期機制
4. **Cloud fallback 會直接 crash** — `mode="auto"` 遇到長 prompt 會路由到 cloud，但 cloud 是 stub，會拋錯
5. **Prompt injection 風險** — 惡意 PDF 內容可以影響 Gemma 的輸出
