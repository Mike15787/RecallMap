# 理解 vs 記憶系統設計 — 研究審視與架構

## 核心問題

使用者學習東西後，系統要記錄其對知識的熟悉程度——「有沒有真正理解」與「有沒有記住」是兩件不同的事，必須分開追蹤。

---

## 理解 vs 記憶：本質差異

| 維度 | 理解（Comprehension） | 記憶（Retention） |
| --- | --- | --- |
| 問的問題 | 「你懂這個概念嗎？」 | 「你還記得這個概念嗎？」 |
| 測量方式 | 解釋、舉例、應用題 | 延遲回憶、填空、無提示提取 |
| 衰減方式 | 不太會隨時間消失，但會「變淺」 | 遵循遺忘曲線，時間越久越難提取 |
| 重新觸發方式 | 需要更深的題型（換角度問） | 需要間隔複習（Spaced Repetition） |

**四種狀態組合：**

- 懂 + 記得 → 真正掌握 🟢
- 懂 + 忘了 → 複習一次就會回來（最常見）🟠
- 不懂 + 記得 → 死背，考試後馬上崩潰
- 不懂 + 忘了 → 真正的盲點，需要重新學 🔴

## 資料結構設計

```python
MasteryRecord {
  concept_id: str

  # ── 理解軸 ──
  comprehension_score: float      # 0.0 ~ 1.0
  comprehension_level: enum       # none / surface / deep / transferable
  last_comprehension_test: datetime
  comprehension_history: List[ComprehensionEvent]

  # ── 記憶軸 ──
  retention_score: float          # 0.0 ~ 1.0
  sm2_interval: int               # 下次複習距今幾天
  sm2_repetitions: int
  last_retention_test: datetime
  next_review_due: datetime
  retention_history: List[RetentionEvent]
}
```

### ComprehensionEvent（理解事件）

```python
ComprehensionEvent {
  timestamp: datetime
  question_type: enum   # explain / apply / analogy / debug
  user_answer: str
  gemma_verdict: enum   # no_understanding / partial / solid / deep
  gemma_reasoning: str
  score_delta: float
}
```

理解分數更新規則（初版）：

- `deep` → +0.3，comprehension_level 升一級
- `partial` → +0.05，不升級
- `no_understanding` → -0.15，標記需要重新解釋

### RetentionEvent（記憶事件）

```python
RetentionEvent {
  timestamp: datetime
  question_type: enum   # cloze / multiple_choice / true_false
  response_quality: int # SM-2：0–5（0=完全忘了，5=秒答）
  new_interval: int
  new_easiness: float
}
```

## 出題邏輯

```
使用者開啟複習 session
        ↓
系統查：comprehension_score < 0.6？
  ├─ 是 → 先出「理解型題目」（解釋、應用、類比）
  │        → Gemma 4 判斷理解層次 → 更新 comprehension_score
  └─ 否 → 檢查 next_review_due <= today？
            ├─ 是 → 出「記憶型題目」（填空、選擇）
            │        → SM-2 更新 interval + next_review_due
            └─ 否 → 這個概念暫時不用複習，跳過
```

## Gemma 4 理解判斷 Prompt 設計

```
概念：{concept}
標準定義：{definition}
使用者的回答：{user_answer}
題目要求：{question}

請評估這個回答屬於哪個理解層次：
- no_understanding：完全錯誤，或只是背出定義但沒有理解
- partial：方向對，但有關鍵缺漏或誤解某個細節
- solid：正確理解核心概念，能說清楚是什麼
- deep：能舉例、類比、或說明這個概念「為什麼這樣設計」

輸出 JSON：
{
  "level": "...",
  "missing_part": "（partial 時填）",
  "next_question_hint": "（建議下一題考什麼角度）"
}
```

## 研究審視（2025-04-10）

### ✅ 設計正確的地方

**理解 vs 記憶分開追蹤**

知識整合進既有 schema 後，對遺忘的抵抗力更強（Sweller et al., 2011）。兩者相關但不等同，分開記錄方向正確。

**SM-2 做記憶追蹤**

Ebbinghaus 的遺忘曲線研究在 2015 年被重新複製，結果相似。最佳複習時間點為：初次學習後 1 小時內、24 小時內、一週後、一個月後（Wollstein & Jabbour, 2022）。SM-2 邏輯與此吻合。

**用作答品質判斷理解（非僅對錯）**

需要生成（production）的測驗比辨識（recognition）測驗對長期記憶的促進效果更好（Roediger & Karpicke, 2006）。

---

### ⚠️ 需要修正的地方

#### 問題一：流暢幻覺（Fluency Illusion）風險

**研究依據**：當 AI 把概念解釋得清晰流暢時，學習者容易誤把「好懂」當作「真的懂了」，即便無法獨立重現或應用內容（Fluency Illusion Review, 2026）。

**對 RecallMap 的衝擊**：使用者看完 Gemma 4 的概念解釋後，用幾乎相同的話回答，AI 可能誤判為 `solid`，但那只是短暫記憶的複製。

**修正方案**：理解型題目設計必須確保**無法直接從剛看過的解釋複製答案**——換情境問、要求舉自己的例子、或與其他概念對比。

---

#### 問題二：理解分數不應只往上累積

**研究依據**：「理解的幻覺」（Illusion of Knowing）在大學生中很普遍，前瞻性判斷的誤差比回顧性判斷更大（Avhustiuk et al., 2018）。同一 session 內的作答表現因短期記憶殘留會被高估。

**修正方案**：加「24 小時延遲確認機制」——當天作答好只算「初步確認」，要等隔天延遲測驗通過，才能真正升級到 `deep` 或 `transferable`。

```python
# 修正後的升級條件
if verdict == "deep" and is_delayed_test:  # 距上次學習 > 24hr
    comprehension_level = upgrade(comprehension_level)
else:
    comprehension_score += 0.1  # 只加分，不升 level
```

---

#### 問題三：測驗題型順序應以生成型優先

**研究依據**：需要「生成」的測驗（簡答、填空）對長期記憶的促進效果比「辨識」型（選擇題、是非題）更強（Roediger & Karpicke, 2006；Greving & Richter, 2018）。

**修正後的出題邏輯**：

| comprehension_score | 題型策略 |
| --- | --- |
| < 0.4（完全不懂） | 先填空/選擇確認基本知識點，再出簡答 |
| 0.4–0.7（部分理解） | 簡答為主，要求舉例或類比 |
| > 0.7（接近掌握） | 換情境的應用題，驗證能否 transfer |

---

#### 問題四：缺少 Interleaving（交錯練習）

**研究依據**：Bjork（1994）指出，分塊練習（blocking）短期感覺學得扎實，但交錯練習（interleaving）帶來更好的長期記憶保留。交錯練習讓學習者區分概念類型而非套用固定模式（Bjork & Bjork, 2011）。

**修正方案**：記憶型複習 session 中，把同一主題下 3–5 個相關概念**交錯出題**，而不是逐一測完再換下一個。

例：「樹的走訪」session 裡，前序、中序、後序混合出現，不是先做完前序才做中序。

---

#### 問題五：理解分數應與記憶分數輕微連動

**研究依據**：記憶是主動重建的過程，不是精確複製（Bartlett, 1932）。長期無法提取的知識，理解也可能退化成表面理解，特別是技術性或高度抽象的概念。

**修正方案**：當記憶測驗未通過時，理解分數應輕微連動下降。

```python
# 記憶測驗失敗時
if response_quality < 2:  # SM-2 判定為遺忘
    comprehension_score *= 0.95  # 輕微衰減，非歸零
```

---

## 修正後設計摘要

| 原設計 | 研究依據的修正建議 |
| --- | --- |
| 理解分數可以當場升級 | 加「24 小時延遲確認」，才能真正升 level |
| 題型由簡到難（選擇→簡答） | 應由生成型優先（簡答→應用），選擇題只用於初期 |
| 概念逐一測驗 | 加入 interleaving，相關概念交錯出題 |
| 理解分數不隨時間衰減 | 記憶分數衰減時，理解分數應輕微連動下降（×0.95） |
| 無流暢幻覺防護 | 理解題設計確保無法直接複製剛讀到的解釋作答 |
