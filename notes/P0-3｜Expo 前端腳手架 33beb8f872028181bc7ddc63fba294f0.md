# P0-3｜Expo 前端腳手架

# 功能描述

在 `apps/mobile/` 建立可以跑的 Expo 專案，設定好開發環境、路由結構、API 連線，讓後續所有前端功能都在這個基礎上開發。

---

# 輸入 / 輸出

|  | 說明 |
| --- | --- |
| **輸入** | 空的 `apps/mobile/` 目錄 |
| **輸出** | 可執行的 Expo 專案，能在 web 模式跑起來並呼叫後端 API |

---

# 實作範圍

## ✅ 包含

- `npx create-expo-app` 建立專案（TypeScript template）
- 安裝 Expo Router（file-based routing）
- 安裝 NativeWind（Tailwind CSS for RN）
- 安裝 Zustand（狀態管理）
- 設定環境變數（`EXPO_PUBLIC_API_URL`）
- 建立基本目錄結構（`app/`、`src/components/`、`src/stores/`、`src/hooks/`）
- 建立一個空白首頁（`app/index.tsx`），顯示 RecallMap 字樣
- 首頁呼叫 `GET /health` 並顯示後端狀態（連線正常 / 無法連線）
- 設定 `package.json` scripts：`start`、`web`、`ios`、`android`
- 更新根目錄 `COMMANDS.md` 加入前端啟動指令

## ❌ 不包含

- 任何業務邏輯頁面（上傳、地圖、對話）
- iOS / Android 實機測試設定（EAS）
- Tauri 桌面整合
- 完整的導覽結構（等 P0-5、P0-6 完成再建）
