# P0-5｜前端 — 上傳介面

# 功能描述

讓使用者能在 web 介面上傳學習材料（PDF、圖片、ChatGPT JSON），呼叫後端 ingest API，並看到上傳結果。這是整個流程的起點。

---

# 輸入 / 輸出

|  | 說明 |
| --- | --- |
| **使用者輸入** | 選擇檔案（PDF / 圖片 / JSON）或輸入 Notion page ID |
| **畫面輸出** | 上傳進度、完成後顯示「已讀取 N 段內容」 |
| **API 呼叫** | `POST /v1/sessions/{id}/ingest`（multipart） |

---

# 實作範圍

## ✅ 包含

- 建立 session（若無現有 session，自動建立）
- 檔案選擇 UI（支援多種格式）
- 上傳進度顯示（loading 狀態）
- 上傳成功：顯示「已讀取 {chunks_added} 段內容」
- 上傳失敗：顯示友善錯誤訊息（不顯示技術錯誤）
- Notion page ID 輸入欄位
- 支援格式：`.pdf`、`.pptx`、`.docx`、`.jpg`、`.png`、`.json`
- 上傳完成後顯示「分析盲點」按鈕，導向學習地圖頁

## ❌ 不包含

- 拖曳上傳（Hackathon 階段用按鈕選擇即可）
- 多檔案同時上傳
- 上傳歷史記錄列表
- 檔案預覽
- source_hint 手動選擇（自動偵測）

---

# UI 文字規範（零門檻語言）

| 情境 | 文字 |
| --- | --- |
| 上傳中 | 「AI 正在讀你的筆記...」 |
| 上傳成功 | 「讀完了！找到 {n} 段學習內容」 |
| 格式不支援 | 「目前支援 PDF、圖片、ChatGPT 或 Gemini 的對話紀錄」 |
| 連線失敗 | 「暫時連不上，請確認後端是否啟動」 |
