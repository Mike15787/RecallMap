## 0. 範圍與非目標

### 範圍
- 桌面單一 app（Tauri + React + Vite + TypeScript + shadcn/ui + Tailwind）
- 涵蓋三條魔法時刻 Demo：
  1. 拖檔 → 地圖 → Calendar 排程
  2. ChatGPT Takeout → 熟悉度排序 → 首題
  3. 貼 ChatGPT 分享連結 → 補盲點

- Onboarding 流程（直接看到拖曳區）
- 蘇格拉底對話（`engine/dialogue.py` 已廢棄）
- 延遲確認機制 UI（`delayed_confirmation.py` P2，UI 可預留 `pending_confirmation` 標記，但不做完整流程）
- 意圖層 Snoozed / Archived 完整操作（P2，UI 可預留 menu 項但可暫時 disabled）
- 多 profile / 多使用者身份驗證
- 行動裝置、Web、iOS、Android（整個 `apps/mobile/` 已刪）

## 4. 功能需求（對應後端 API）

後端路由依技術規格書 §5.3 與 2026-04-19 筆記調整為 `/v1/capsules/...`。

### 4.1 Capsule 管理
| 功能 | API | UI 元件 |
|---|---|---|
| 列出 capsule | `GET /v1/capsules` | `CapsuleSwitcher` / `HomePage` |
| 建立 capsule | `POST /v1/capsules`（body：name + 選填 exam_date） | `CreateCapsuleDialog` |
| 取得單一 capsule + mastery summary | `GET /v1/capsules/:id` | `CapsulePage` 初始載入 |
| 更新 name / exam_date | `PATCH /v1/capsules/:id` | `CapsulePage` header 編輯 |
| 刪除 | **不提供**（符合規範） | — |

### 4.2 Ingest
| 功能 | API | UI 元件 |
|---|---|---|
| 上傳檔案（PDF/PPTX/DOCX/圖片/JSON） | `POST /v1/capsules/:id/ingest`（multipart） | `DropZone` |
| 前端檔案驗證 | 本地 | `lib/validators.ts`（PDF ≤ 50MB、單次圖片 ≤ 20 張） |
| ChatGPT 分享連結爬取 | `POST /v1/capsules/:id/ingest-url` | `UrlIngestInput`（URL 格式即時驗證） |
| Ingest 進度 / 狀態 | `GET /v1/capsules/:id/ingest/:jobId` 或 WebSocket（若有） | `IngestProgress`（大檔同步處理要顯示 Loading） |
| 錯誤提示 | 解析 4xx / 5xx | `Toast`：掃描版 PDF、密碼保護、格式錯誤、超出上限 |

**支援格式顯示**：`.pdf .pptx .docx .jpg .png .webp .json`（ChatGPT / Gemini Takeout）

### 4.3 Concept 過濾
| 功能 | API | UI 元件 |
|---|---|---|
| 取得 ingest 後的 concept 清單 | `GET /v1/capsules/:id/concepts?status=pending` | `ConceptCheckList` |
| 勾選要學的 concept | `PATCH /v1/concepts/bulk`（body：id + include flag） | `ConceptFilterPage` |
| 標記為已知（邊緣化） | `PATCH /v1/concepts/:id/intent { intent: "known" }` | Map 右鍵選單 |

### 4.4 雙軸學習地圖
| 功能 | API | UI 元件 |
|---|---|---|
| 取得 Comprehension Map 資料 | `GET /v1/capsules/:id/map?axis=comprehension` | `LearningMap` |
| 取得 Retention Map 資料 | `GET /v1/capsules/:id/map?axis=retention` | `LearningMap` |
| 節點詳情（concept 解釋 + chunks + 鄰居） | `GET /v1/concepts/:id/detail` | `ConceptSidePanel` |
| 觸發 concept quiz | `POST /v1/concepts/:id/quiz` → 取得 quizSessionId | 右鍵「做這個概念的題」 |

**節點樣式規則**：
- 顏色：四狀態 🔴 blind / 🟡 fuzzy / 🟠 needs_review / 🟢 known
- ✳️ 標記：`pending_confirmation=true`（UI 預留；P2 才真的進延遲確認流程）
- 透明度：`intent=snoozed` 50%、`archived` 隱藏（Retention Map 不出現）
- 邊（edge）：embedding cosine > 0.75 連線；粗細可隨相似度微調

### 4.5 Quiz Session
| 功能 | API | UI 元件 |
|---|---|---|
| 建立 quiz session（「今天的複習」） | `POST /v1/session/next`（回傳 quizSessionId + 題目清單） | `HomePage` 大按鈕 |
| 單題取得 | 由 session payload 提供 | `QuestionRenderer` |
| 送出答案 | `POST /v1/quiz/:quizSessionId/answers`（body：quiz_id + answer） | 各題型元件 |
| 結束 session | `POST /v1/quiz/:quizSessionId/finish` | `QuizSummary` |

**題型渲染分發**（依 `domain` + `question_type`）：

| Domain | Type | 元件 |
|---|---|---|
| any | `short_answer` | `ShortAnswerQuestion` |
| any | `multiple_choice` | `MultipleChoiceQuestion` |
| any | `true_false` | `TrueFalseQuestion` |
| any | `fill_blank`（通用） | `FillBlankQuestion` |
| `math_formula` | `fill_blank` | `FormulaFillQuestion`（使用 `formula_tokens`） |
| `math_formula` | `short_answer` | `ShortAnswerQuestion` + `SymbolPalette` |
| `programming` | `fill_blank` | `CodeFillQuestion`（monospace + token 按鈕） |
| `language` | `match_pairs` | `MatchPairsQuestion` |
| `memorization` | `sort_items` | `SortItemsQuestion` |
| `memorization` | `image_label` | `ImageLabelQuestion`（讀 `image_ref`） |
| `memorization` | `true_false × 5` | `BatchTrueFalseQuestion` |
| `calculation` | `fill_blank` | `FillBlankQuestion` + `numeric_tolerance` 提示 |

**Quiz 流程**：
1. 顯示題目 → 使用者作答 → 送出
2. 顯示即時回饋（score + feedback + 更新後 comprehension/retention）
3. 「下一題」按鈕 → 下一道（Interleaving：多 concept 交錯）
4. 全部答完 → `QuizSummary`（總結本次學了多少、哪些 concept 升級、下次複習日）

### 4.6 排程
| 功能 | API | UI 元件 |
|---|---|---|
| 取得 SM-2 複習時程 | `GET /v1/capsules/:id/schedule` | `ReviewTimeline` |
| 授權 Google Calendar | `GET /v1/integrations/calendar/auth` → 瀏覽器跳轉 | `CalendarAuthCard` |
| 寫入 Calendar events | `POST /v1/capsules/:id/schedule/calendar` | 一鍵按鈕 |
| 匯出 .ics | `GET /v1/capsules/:id/schedule.ics`（下載） | `IcsExportButton` |

### 4.7 系統
| 功能 | API | UI 元件 |
|---|---|---|
| LLM 健康檢查 | `GET /v1/llm/health`（已實作） | Sidebar 燈號 + Settings 詳細 |
| Notion token 設定 | `PUT /v1/settings/notion-token` | `SettingsPage` |
| 全域錯誤 | — | `ErrorBoundary` + `Toast` |

## 6. 型別規範

### 6.1 位置
- 所有型別放 `apps/desktop/src/types/`
- **不再** 依賴 `packages/shared`（已刪）
- 與後端 Pydantic schema 人工對齊；未來可評估 `openapi-typescript` 自動產生

### 6.2 命名
- 介面以 `I` 前綴：`ICapsule`, `IConcept`, `IQuiz`, `IMasteryRecord`, `IComprehensionEvent`, `IRetentionEvent`
- 列舉與 union：`QuestionType`, `QuizStrategy`, `QuizDomain`, `ConceptIntent`

### 6.3 核心型別（摘錄）
```ts
// 對應後端 dataclass
export interface ICapsule {
  capsule_id: string
  name: string
  exam_date: string | null          // ISO 8601 或 null
  created_at: string
  mastery_summary: {
    blind_count: number
    fuzzy_count: number
    review_due_count: number
    known_count: number
  }
}

export interface IConcept {
  concept_id: string
  name: string
  summary: string
  tags: string[]                    // 固定 3 個
  source_chunk_ids: string[]
  capsule_ids: string[]             // 跨 capsule 共用時列多個
  mastery: IMasteryRecord
}

export interface IMapNode {
  node_id: string                   // = concept_id
  concept: string
  zone: 'blind' | 'fuzzy' | 'needs_review' | 'known'
  comprehension_score: number
  retention_score: number
  pending_confirmation: boolean
  next_review_due: string | null
  intent: 'active' | 'snoozed' | 'archived'
  snooze_until: string | null
}

export interface IMapEdge {
  source: string
  target: string
  similarity: number                // 0.75+
}

export interface IQuiz {
  quiz_id: string
  capsule_id: string
  concept: string
  domain: QuizDomain
  question_type: QuestionType
  question: string
  options?: string[]
  formula_tokens?: string[]
  symbol_palette?: string[]
  code_snippet?: string
  image_ref?: string
  sort_items?: string[]
  match_pairs?: [string, string][]
  numeric_tolerance?: number
  hint?: string
}
```

## 7. UI / UX 規範

### 7.1 語言
- UI 一律**英文**
- 錯誤訊息避免技術術語（參考技術規格書 §10）：
  - ❌ "Embedding vector..."  → ✅ "Reading your notes..."
  - ❌ "Token limit exceeded" → ✅ "This note is long, splitting into chunks"
  - ❌ "API error"             → ✅ "Temporarily unreachable, please retry"

### 7.2 顏色規範（Tailwind）
| 狀態 | Tailwind class |
|---|---|
| 🔴 blind | `bg-red-500` / `text-red-400` |
| 🟡 fuzzy | `bg-yellow-500` / `text-yellow-400` |
| 🟠 needs_review | `bg-orange-500` / `text-orange-400` |
| 🟢 known | `bg-green-500` / `text-green-400` |
| ✳️ pending | `ring-2 ring-blue-400` |

### 7.3 Dark mode
預設 dark theme（既有 walking-skeleton 已如此）。不強制 light mode；未來視需求再加 toggle。

### 7.4 回饋延遲
- 所有非同步操作 < 200ms 無需 loading
- 200ms – 2s 顯示 Skeleton 或行內 spinner
- > 2s 顯示 `LoadingOverlay` + 進度條 + 友善文案

### 7.5 鍵盤操作（選配）
- Quiz：`Enter` 送出、`1-4` 選擇題快捷、`Esc` 退出
- Map：方向鍵移動視角、`F` 聚焦選中節點

## 8. 狀態管理策略

| 狀態種類 | 放哪裡 |
|---|---|
| 伺服器資料（capsules、concepts、map、quiz） | TanStack Query cache |
| 當前選中的 capsule id | Zustand（`capsule-store`） |
| Quiz session 進行中的暫存答案、當前題號 | Zustand（`quiz-store`） |
| 單一表單輸入 | `useState` / react-hook-form |
| UI 開關（dialog 開合） | 元件內 `useState` |

**原則**：伺服器狀態交給 TanStack Query；client-only 狀態才放 Zustand；避免把 query 結果再寫進 Zustand。

## 9. 錯誤處理

- API client 層統一拋 typed error：`ApiError { status, code, message }`
- TanStack Query `onError` → `toast.error()`
- 頂層 `ErrorBoundary` 包住 Router，元件 crash 顯示友善畫面
- Ingest 錯誤（密碼保護、掃描版 PDF、格式錯誤）→ 內聯顯示在 DropZone 附近
- 404（capsule 不存在）→ 顯示 EmptyState + 回首頁按鈕
